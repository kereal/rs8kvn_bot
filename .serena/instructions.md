## Во время работы

### Общение
- **Всегда пиши на русском языке** (кроме кода, коммитов и технических терминов)
- Будь кратким, но информативным
- Используй markdown для форматирования
- Приветствуются emoji для визуализации

### Код
- Следуй существующему стилю кода
- Добавляй комментарии на русском для сложной логики
- Пиши тесты для новой функциональности
- Проверяй что существующие тесты проходят

### Git (СТРОГО!)
- Используй conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`
- Коммить часто, малыми изменениями
- ❌ **НИКОГДА не коммить напрямую в `dev` или `main`**
- ✅ **ВСЕГДА: создать ветку → commit → push → PR в `dev`**
- Ветвление: `docs/`, `fix/`, `feature/` — выбирается по типу изменения
- После релиза — синхронизировать `dev` с `main`

### Terminal tool

Инструмент `terminal` выполняет shell-команды в контексте проекта.

**Параметр `cd`:** Всегда используй `cd: "."` — это корень активного проекта.

**Важно:**
- `.` = корень проекта (правильно)
- `/home/kereal/rs8kvn_bot` — абсолютные пути НЕ работают, вызывают worktree error
- Команда выполняется в новом shell-процессе, состояние между вызовами не сохраняется

**Для долгих команд** (build, test, lint) — указывай `timeout_ms`:
```
timeout_ms: 120000  # 2 минуты
```

**НЕ запускай:**
- Серверы без `timeout_ms` (`go run`, `npm run dev` — зависнут)
- Команды с shell-подстановками (`$VAR`, `${VAR}`, `$(...)`) — подставляй значения вручную
- `cd` внутри команды — используй параметр `cd` инструмента

## Важные ограничения

### Архитектура
- **3x-ui = 1 сервер** — это ограничение архитектуры
- Для многосерверности нужен кастомный генератор подписок
- SQLite подходит до ~500 пользователей

### Приоритеты
- **Сейчас:** Монетизация и рост (платежи, промокоды)
- **Не сейчас:** Enterprise-фичи (ML, блокчейн, P2P)
- У пользователя ~10 клиентов — фичи для малого бизнеса

### Документация
- **На русском:** все объяснения, комментарии в памяти
- **На английском:** код, коммиты, технические термины
