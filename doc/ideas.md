## 1. Автоматические уведомления о лимите трафика

**Приоритет:** Высокий  
**Время реализации:** 2-3 часа  
**Сложность:** Низкая

### Описание
Когда пользователь израсходовал 80%, 90%, 100% трафика — бот автоматически отправляет уведомление.

### Пример сообщения
```
⚠️ Вы израсходовали 80% трафика (24 из 30 ГБ)
📅 Обновление через 5 дней
💡 Подключите друга и получите +5 ГБ бонуса
```

### Зачем нужно
- Пользователи не мониторят трафик сами
- Узнают о лимите когда уже всё сгорело
- Снижает количество обращений в поддержку

### Техническая реализация
- Cron-задача запускается каждый час
- Проверяет всех активных подписчиков через 3x-ui API
- Фильтрует тех, кто достиг порога (80/90/100%)
- Отправляет уведомление один раз за период (флаг в БД)

### Файлы
- `internal/notifier/traffic_alert.go` (новый)
- `internal/database/database.go` — добавить поле `traffic_alert_sent`
- `cmd/bot/main.go` — запустить cron-задачу

## 2. Система наград за рефералов

**Приоритет:** Высокий  
**Время реализации:** 3-4 часа  
**Сложность:** Средняя

### Описание
Геймификация реферальной системы с уровнями и наградами.

### Уровни
| Уровень | Рефералов | Награда |
|---------|-----------|---------|
| 🥉 Бронза | 1-3 | +5 ГБ трафика |
| 🥈 Серебро | 4-10 | +10 ГБ + продление на 7 дней |
| 🥇 Золото | 11-25 | Безлимит на месяц |
| 💎 Платина | 26+ | VIP-статус, приоритетная поддержка |

### Пример сообщения
```
🎉 Поздравляем! Вы достигли уровня Серебро!
🎁 Ваши награды:
  • +10 ГБ трафика
  • Продление подписки на 7 дней

👥 У вас 7 рефералов
🔥 До уровня Золото осталось 4 реферала
```

### Зачем нужно
- Мотивация приглашать друзей
- Виральный рост user base
- Повышение вовлечённости

### Техническая реализация
- Таблица `referral_rewards` в БД
- Проверка уровня при каждом новом реферале
- Автоматическое применение наград
- Уведомление пользователя о достижении уровня

## 3. Интеграция с ЮKassa/Т-Банк Pay

**Приоритет:** Высокий  
**Время реализации:** 4-6 часов  
**Сложность:** Средняя

### Описание
Автоматическая оплата подписки через платёжные системы.

### Тарифы
```
💳 Оплатить подписку
├─ 1 месяц — 299₽
├─ 3 месяца — 799₽ (скидка 11%)
└─ 12 месяцев — 2499₽ (скидка 30%)
```

### Flow
1. Пользователь выбирает тариф
2. Переходит на страницу оплаты (ЮKassa)
3. После успешной оплаты — webhook на бэкенд
4. Автоматическое создание/продление подписки в 3x-ui

### Зачем нужно
- Монетизация проекта
- Пассивный доход
- Автоматизация вместо ручной обработки

### Техническая реализация
- Интеграция с ЮKassa API или Т-Банк Pay API
- Webhook endpoint `/payment/callback`
- Таблица `payments` в БД
- Автоматическое создание подписки после оплаты

## 6. Авто-балансировка между серверами

**Приоритет:** Средний  
**Время реализации:** 4-5 часов  
**Сложность:** Средняя

### Описание
Распределение нагрузки между несколькими 3x-ui серверами.

### Алгоритм
1. Периодическая проверка нагрузки на каждый сервер
2. Новых пользователей направлять на наименее загруженный
3. При перегрузке (>80% CPU) — миграция части пользователей

### Конфигурация
```env
XUI_SERVERS=server1.example.com,server2.example.com,server3.example.com
XUI_LOAD_BALANCE=strategy=least_connections
```

### Зачем нужно
- Нет единой точки отказа
- Равномерная нагрузка
- Автоматическое масштабирование

## 8. Реферальная программа для админов других ботов

**Приоритет:** Низкий  
**Время реализации:** 3-4 часа  
**Сложность:** Средняя

### Описание
API для интеграции с другими ботами/сервисами.

### Схема работы
1. Другой бот регистрируется как партнёр
2. Получает API-ключ и реферальную ссылку
3. Его пользователи подписываются через VPN-бота
4. Партнёр получает комиссию с каждой оплаты

### API Endpoints
```
POST /api/v1/partners/register
GET  /api/v1/partners/stats
GET  /api/v1/partners/referrals
POST /api/v1/subscriptions/create
```

### Зачем нужно
- Партнёрская сеть
- Масштабирование без маркетинга
- Пассивные продажи

## 9. Автоматическая проверка работоспособности VPN

**Приоритет:** Высокий  
**Время реализации:** 2-3 часа  
**Сложность:** Низкая

### Описание
Регулярная проверка доступности серверов.

### Алгоритм
1. Каждый час создавать test-подключение
2. Проверить скорость и latency
3. Если сервер недоступен → алерт админу
4. Автоматический перенос пользователей на резервный сервер

### Уведомление админу
```
🚨 Сервер server1.example.com недоступен!

📊 Последняя проверка: 2 минуты назад
👥 Затронуто пользователей: 45
✅ Автоматически перенесены на server2

🔍 Действия:
   /server_status — детали
   /rollback — вернуть обратно
```

### Зачем нужно
- Прозрачность для админа
- Нет жалоб "не работает"
- Автоматическое восстановление
