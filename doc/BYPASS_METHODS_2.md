## 1. Обзор угроз (что делает правительство)

### 1.1 Методы цензуры в интернете

Правительства используют различные технологии для контроля и ограничения доступа к интернет-ресурсам. В России эти методы активно развиваются с 2019 года.

#### 🔍 DPI (Deep Packet Inspection)

**Что это:**
- Глубокий анализ пакетов на уровне содержимого
- Позволяет определять протоколы, даже если они на нестандартных портах
- Анализирует заголовки и payload пакетов

**Как работает:**

```
┌─────────────────────────────────────────────────────────────┐
│                    DPI-система провайдера                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Входящий пакет ──► Анализ ──► Решение                      │
│                           │                                 │
│                           ├─► SNI inspection                │
│                           ├─► TLS fingerprinting            │
│                           ├─► Protocol detection            │
│                           ├─► Statistical analysis          │
│                           └─► Application Layer detection   │
│                                                             │
│  Результаты анализа:                                        │
│  ✅ LEGITIMATE ──► Пропустить                              │
│  ⚠️ SUSPICIOUS ──► Throttle / Log                          │
│  ❌ BLOCKED ──► Отбросить пакет                            │
└─────────────────────────────────────────────────────────────┘
```

**Методы DPI-анализа:**

| Метод | Описание | Эффективность |
|-------|----------|---------------|
| **SNI inspection** | Чтение доменного имени в TLS ClientHello | ⭐⭐⭐⭐⭐ |
| **TLS fingerprinting** | Анализ сигнатур TLS-рукопожатия | ⭐⭐⭐⭐ |
| **Protocol detection** | Определение протокола по паттернам | ⭐⭐⭐⭐ |
| **Statistical analysis** | Анализ размера и таймингов пакетов | ⭐⭐⭐ |
| **Application Layer detection** | Анализ данных прикладного уровня | ⭐⭐⭐⭐ |

**Оборудование в России:**
- TSPU (Технические средства противодействия угрозам) — установлены у провайдеров
- Платформы DPI от российских и китайских производителей
- Интеграция с СОРМ (Система оперативно-розыскных мероприятий)

**Текущее состояние (2026):**
- Полнота покрытия: ~95% провайдеров
- Глубина анализа: до Layer 7
- Задержка: 2-5 мс на пакет
- Точность детекции: 90-95% для известных протоколов

├────────────────────────────────────────────────────────┤
│                                                        │

├──────────────────────────────────────────────────────────┤

## 2. VPN технологии

### 2.1 Протоколы и их особенности

#### OpenVPN

**Технология:**
- SSL/TLS для шифрования
- Поддержка UDP и TCP
- Возможность использования на любом порту

**Преимущества:**
- ✅ Стабильность и надёжность
- ✅ Широкая совместимость
- ✅ Гибкая настройка
- ✅ Аутентификация сертификатами

**Недостатки:**
- ❌ Легко детектируется DPI
- ❌ Характерный handshake pattern
- ❌ Медленное установление соединения
- ❌ Высокие накладные расходы

**Детекция DPI:**

```
OpenVPN TLS Handshake Detection:
┌────────────────────────────────────────────────────────┐
│  OpenVPN Fingerprint:                                  │
│  ├─► Client Hello с OpenVPN-specific extensions        │
│  ├─► Push, Pull, Control channel packets               │
│  ├─► Статистический анализ размера пакетов             │
│  └─► P_CONTROL_HARD_RESET_CLIENT_V2 pattern            │
│                                                        │
│  DPI detection confidence: 95%+                        │
│  Blocking rate: 99%                                    │
└────────────────────────────────────────────────────────┘
```

**Вердикт:** ❌ Не рекомендуется для обхода цензуры в РФ

#### WireGuard

**Технология:**
- Современный протокол с state-of-the-art криптографией
- ChaCha20-Poly1305 шифрование
- Минимальный код (4k строк)
- Высокая производительность

**Преимущества:**
- ✅ Высокая производительность
- ✅ Малые накладные расходы
- ✅ Быстрое соединение
- ✅ Простая конфигурация

**Недостатки:**
- ❌ Статические заголовки (detectable)
- ❌ Предсказуемый размер пакетов
- ❌ Нет обфускации
- ❌ Легко блокируется по fingerprint

**Fingerprint WireGuard:**

```python
# WireGuard packet detection (DPI perspective)
def is_wireguard(packet):
    """
    WireGuard имеет предсказуемую структуру пакетов:
    - Initiation: 148 bytes, starts with 0x01
    - Response: 92 bytes, starts with 0x02
    - Cookie Reply: 64 bytes, starts with 0x03
    - Transport: 32+ bytes, starts with 0x04
    """
    if len(packet) == 148 and packet[0:4] == b'\x01\x00\x00\x00':
        return True  # WireGuard initiation
    if len(packet) == 92 and packet[0:4] == b'\x02\x00\x00\x00':
        return True  # WireGuard response
    return False
```

**WireGuard Packet Structure:**

```
┌─────────────────────────────────────────────────────────┐
│         WireGuard Handshake Initiation Packet           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Offset  Size  Field                Value               │
│  ─────   ────  ─────                ─────               │
│  0       1     Message Type         0x01                │
│  1       3     Reserved             0x00 0x00 0x00      │
│  4       4     Sender Index         random              │
│  8       32    Ephemeral Public     ChaCha20 key        │
│  40      48    Encrypted Static     AES-256-GCM         │
│  88      56    Encrypted Timestamp  AES-256-GCM         │
│  144     16    MAC1                 Poly1305            │
│                                                         │
│  Total: 148 bytes (IPv4)                               │
│  Total: 168 bytes (IPv6)                               │
│                                                         │
│  ⚠️ FIXED SIZE = Easy to detect!                       │
└─────────────────────────────────────────────────────────┘
```

**Вердикт:** ⚠️ Требует обфускации (AmneziaWG, Warp+)

#### AmneziaWG — Модифицированный WireGuard 🔥

**Что это:**

AmneziaWG — это доработанная версия WireGuard от команды AmneziaVPN, специально разработанная для обхода DPI в России и других странах с цензурой.

**Ключевые отличия от обычного WireGuard:**

```
┌──────────────────────────────────────────────────────────────┐
│              AmneziaWG vs WireGuard                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  WireGuard:                                                  │
│  ├─► Фиксированный размер пакетов (148/92/64/32+ bytes)     │
│  ├─► Предсказуемые байты в начале (0x01, 0x02, 0x03, 0x04)  │
│  ├─► Нет обфускации                                          │
│  └─► Легко детектируется DPI                                 │
│                                                              │
│  AmneziaWG:                                                  │
│  ├─► Переменный размер пакетов (Jc - Junk Count)            │
│  ├─► Случайные байты в начале (маскировка)                   │
│  ├─► Добавление "junk" данных для обфускации                │
│  ├─► Имитация случайного UDP трафика                        │
│  └─► DPI не может определить протокол                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**Технические параметры обфускации:**

| Параметр | Описание | Рекомендуемое значение |
|----------|----------|------------------------|
| **Jc** (Junk Packet Count) | Количество junk-пакетов | 3-10 |
| **Jmin** (Junk Minimum Size) | Минимальный размер junk | 50 |
| **Jmax** (Junk Maximum Size) | Максимальный размер junk | 1000 |
| **S1** (Init Packet Junk Size) | Размер junk в init пакете | 15-150 |
| **S2** (Response Packet Junk Size) | Размер junk в response | 15-150 |
| **H1** (Init Packet Junk Header) | Заголовок junk в init | Случайный |
| **H2** (Response Packet Junk Header) | Заголовок junk в response | Случайный |

**Как работает обфускация:**

```
Обычный WireGuard Handshake:
┌────────────────────────────────────────┐
│ Client ──► [148 bytes, type=0x01] ──► Server │
│ Server ──► [92 bytes, type=0x02] ──► Client │
└────────────────────────────────────────┘
⚠️ Легко обнаружить по размеру и типу

AmneziaWG Handshake:
┌─────────────────────────────────────────────────────────┐
│ Client ──► [Variable size, random header] ──► Server   │
│           ├─► Init packet + random junk data            │
│           └─► Size: 148 + S1 bytes (random)             │
│                                                         │
│ Server ──► [Variable size, random header] ──► Client   │
│           ├─► Response packet + random junk data        │
│           └─► Size: 92 + S2 bytes (random)              │
│                                                         │
│ Additional:                                              │
│ ├─► Jc junk packets sent before/after real data        │
│ └─► All packets look like random UDP traffic            │
└─────────────────────────────────────────────────────────┘
✅ DPI не может отличить от обычного UDP трафика
```

**Преимущества AmneziaWG:**

#### Shadowsocks

**Технология:**
- Прокси-протокол с шифрованием
- Разработан специально для обхода GFW (Great Firewall of China)
- Легковесный и быстрый

**Преимущества:**
- ✅ Хорошая скорость
- ✅ Низкие накладные расходы
- ✅ Поддержка плагинов обфускации
- ✅ Простая настройка

**Недостатки:**
- ⚠️ Vanilla версия обнаруживается в РФ
- ⚠️ Требует плагины (v2ray-plugin, obfs)
- ❌ Не поддерживает UDP в базовой версии

**Конфигурация Shadowsocks:**

```json
{
  "server": "your-server.com",
  "server_port": 8388,
  "password": "your-password",
  "timeout": 300,
  "method": "aes-256-gcm",
  "fast_open": true,
  "nameserver": "8.8.8.8",
  "mode": "tcp_and_udp",
  "plugin": "v2ray-plugin",
  "plugin_opts": "server;tls;host=your-domain.com"
}
```

**Вердикт:** ⚠️ Требует плагины обфускации для России

#### V2Ray / Xray (VLESS, VMess)

**Технология:**
- Мульти-протокольная платформа
- VLESS — легковесный протокол без лишнего шифрования
- VMess — оригинальный протокол V2Ray
- Поддержка различных транспортов (WebSocket, HTTP/2, gRPC, QUIC)

**Преимущества:**
- ✅ Гибкая конфигурация
- ✅ Множество транспортов
- ✅ Активное развитие
- ✅ Reality поддержка

**Недостатки:**
- ⚠️ Сложная настройка
- ⚠️ VMess обнаруживается в РФ (без обфускации)

**Вердикт:** ✅ VLESS+Reality — лучший выбор

#### Trojan

**Технология:**
- Маскировка под HTTPS трафик
- Использует стандартный TLS
- Простая архитектура

**Преимущества:**
- ✅ Хорошая маскировка
- ✅ Простая конфигурация
- ✅ Стандартный TLS

**Недостатки:**
- ⚠️ Требует домен и сертификат
- ⚠️ Может детектироваться по поведению

**Конфигурация Trojan:**

```json
{
  "run-type": "client",
  "local-addr": "127.0.0.1",
  "local-port": 1080,
  "remote-addr": "your-server.com",
  "remote-port": 443,
  "password": ["your-password"],
  "ssl": {
    "verify": true,
    "sni": "your-server.com"
  }
}
```

**Вердикт:** ⚠️ Хорошо, но VLESS+Reality лучше
