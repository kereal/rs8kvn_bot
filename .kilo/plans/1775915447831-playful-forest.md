# Plan: Aggressive Safe Code Optimisation Refactoring

## Goal
Refactor provided scripts into shortest, fastest, leanest idiomatic drop-in replacements while preserving **100%** of all observable behaviour:
- ✅ All public inputs/outputs preserved exactly
- ✅ All side effects, error behaviours, runtime ordering unchanged
- ✅ Licence headers, copyright, attribution retained completely
- ✅ Intent-bearing comments preserved
- ✅ No semantic changes whatsoever

---

## Workflow Steps

### 1. **Input Handling**
- User will attach script files one at a time
- For each file first verify complete original content
- Document all observable behaviours before any changes

### 2. **Refactoring Principles (Strict Order)**
1.  **Preservation first**: Map every public API, return value, error case, timing, and side effect before touching code
2.  **Collapse redundancy**: Inline trivial wrappers, remove unnecessary abstraction layers, eliminate superfluous intermediate variables
3.  **Standard library maximisation**: Replace manual implementations with builtin idioms, vectorised operations, and standard library functions
4.  **Safe micro-optimisations**: Apply loop fusion, conditional merging, short-circuiting, and constant folding where behaviour is identical
5.  **Accepted clever hacks**: Use portable, deterministic, widely known optimisations only (no version-dependent tricks)
6.  **Minimal documentation**: Keep only comments explaining non-obvious optimisations, strip all noise
7.  **Self-containment**: No code movement across files, no new modules

### 3. **Verification Check List**
Before finalising each refactored file confirm:
- [ ] Identical output for all valid inputs
- [ ] Identical error conditions and error messages
- [ ] Identical side effect ordering
- [ ] No new external dependencies
- [ ] All original licence/attribution intact
- [ ] No behaviour changes at all, only implementation changes

### 4. **Output Format**
For each script:
1.  Full refactored file content first
2.  Concise precise list of optimisations applied with safety justification

---

## Constraints
- ❌ No semantic changes allowed. Ever.
- ❌ No new external dependencies
- ❌ No code moved between files
- ❌ No fragile or version-specific optimisations
- ✅ All existing behaviour is sacred
