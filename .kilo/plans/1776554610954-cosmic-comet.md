# Plan: Fix godoclint issues in metrics.go

## Problem

The package fails godoclint because exported vars in the var block (lines 12-140) lack symbol-leading comments. Currently there are group headers like `// HTTP metrics`, `// Bot metrics`, but each individual exported variable doesn't have its own GoDoc comment starting with the symbol name.

## Solution

Add individual GoDoc comments immediately above each exported variable, describing their purpose and labels. Convert from grouped block to individually commented declarations.

## Changes Required
